## 1. プロジェクト目的
Minecraft（Paper）内のプレイヤー間の距離と方向に基づき、ブラウザ上でリアルタイムに減衰・定位（パンニング）処理された音声通話を提供する。

---

## 2. システム全体図
システムの構成要素を3つのセグメントに分けます。

### A. Paperプラグイン (Java/Kotlin)
* **座標取得:** ログイン中のプレイヤーの座標 $(x, y, z)$ と向いている方向（Yaw）を一定間隔で取得。
* **認証紐付け:** プレイヤーがWebサイトにアクセスするための「ワンタイムトークン」を生成・発行。
* **データ送信:** 取得した座標データを外部DB（Redis等）へ高速に書き込む。

### B. データ基盤 (Redis / Web API)
* **高速I/O:** 毎秒数回更新される全プレイヤーの座標を保持。
* **セッション管理:** ゲーム内のUUIDとWeb側のセッション情報を紐付け。

### C. Webアプリケーション (Node.js / React / WebRTC)
* **認証:** ゲーム内で発行されたトークンを用いて、ブラウザ上で本人確認。
* **音声通信 (WebRTC):** * SFU（Media Server）を介した音声データの送受信。
    * **重要:** ユーザーが「サーバー側でMix」を希望されていますが、遅延とサーバー負荷を考慮し、**「演算（計算）はサーバー、音響処理（減衰）は各クライアントのブラウザ」**で行うハイブリッド方式を推奨します。
* **空間音響処理:** 受信した他プレイヤーの座標と自分の座標から、Web Audio APIを用いて音量と定位をリアルタイム変更。

---

## 3. 機能要件 (Functional Requirements)

| カテゴリ | 要求機能 | 詳細 |
| :--- | :--- | :--- |
| **ゲーム内** | `/vc join` コマンド | ブラウザ接続用のURLとURLトークンを表示する。 |
| **位置同期** | 低遅延座標更新 | 200ms〜500ms間隔でDBの座標を更新する。 |
| **Web認証** | プレイヤー照合 | 入力されたトークンが有効かDBで照合し、UUIDを取得する。 |
| **音声処理** | 距離減衰 | 距離 $d$ に応じて音量を $1/d$ 等の関数で減衰させる。 |
| **音声処理** | 3Dパニング | プレイヤーの向き（Yaw）に基づき、左右のスピーカー出力を調整する。 |
| **UI/UX** | ステータス表示 | Web画面上で「現在接続中の近くのプレイヤー」を一覧表示する。 |

---

## 4. 技術仕様 (Technical Specifications)

### 距離減衰の計算式
基本的には以下の3次元距離の公式を用いますが、Web Audio APIの `PannerNode` を使えば、内部的にこの計算を自動化できます。

$$d = \sqrt{(x_2 - x_1)^2 + (y_2 - y_1)^2 + (z_2 - z_1)^2}$$

### 通信プロトコル
* **プラグイン ↔ DB:** Redis (Jedis/Lettuce) または WebSocket
* **ブラウザ ↔ Webサーバー:** WebSockets (Socket.io等) で座標を受信
* **ブラウザ ↔ ブラウザ:** WebRTC (Mediasoup または LiveKit 等のSFUを推奨)

---

## 5. 非機能要件 (Non-Functional Requirements)
* **低遅延:** 音声の遅延は 200ms 以内を目標とする。
* **スケーラビリティ:** 同時に30人程度が会話してもサーバーがダウンしない。
* **セキュリティ:** トークンは使い捨てとし、他人が勝手に他人のIDで入ることを防ぐ。
