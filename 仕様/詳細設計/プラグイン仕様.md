## 1. プラグインの内部構造（アーキテクチャ）
プラグインの責務を3つのコンポーネントに分離します。
| コンポーネント | 役割 | 処理頻度 |
|---|---|---|
| **Data Collector** | 全プレイヤーの座標、Yaw、ワールド、使用アイテム、権限を収集。 | 5 Ticks (250ms) |
| **Redis Publisher** | 収集したデータをJSON化し、Jedisを使用してRedisへ非同期送信。 | 5 Ticks (250ms) |
| **Command Executor** | /vc および /radio コマンドの処理、Django APIとの通信。 | 随時（ユーザー操作時） |
## 2. 座標送信ロジックの詳細設計
### 2.1 送信データのフィルタリング
全プレイヤーのデータを無条件に送るのではなく、**「ボイスチャットに接続中（Webログイン済み）」**のプレイヤーのみに限定するとさらに軽量化できますが、最初はシンプルに「全オンラインプレイヤー」を対象にします。
### 2.2 非同期パイプライン
メインスレッド（ゲームの計算用スレッド）でRedisと通信するのは厳禁です。
```java
// イメージ：非同期タスクでのRedis送信
new BukkitRunnable() {
    @Override
    public void run() {
        try (Jedis jedis = jedisPool.getResource()) {
            Pipeline pipeline = jedis.pipelined();
            for (Player player : Bukkit.getOnlinePlayers()) {
                String key = "vchat:player:" + player.getUniqueId();
                String data = buildJson(player); // JSON構築
                pipeline.setex(key, 5, data);    // TTL 5秒でセット
            }
            pipeline.sync();
        }
    }
}.runTaskTimerAsynchronously(plugin, 0L, 5L); // 5 Ticks間隔

```
## 3. コマンド仕様（Command API）
ユーザーがゲーム内からWebへスムーズに遷移するためのインターフェースです。
### /vc join (または /vc link)
 1. **内部処理:** Djangoの /api/vc/token/generate/ へプレイヤーのUUIDを送信。
 2. **レスポンス:** 6桁のワンタイムトークンを受け取る。
 3. **表示:** プレイヤーにクリック可能なメッセージを表示。
   * 「[VC接続] ここをクリックしてログイン（コード: 123456）」
   * URL例: https://vc.clusters-prj.com/login?token=123456
### /radio <channel>
 1. **内部処理:** プレイヤーのメタデータ radio_channel を更新。
 2. **即時反映:** 次のRedis送信パケットにこのチャンネルIDが含まれる。
 3. **演出:** チャンネル切り替え時に「カチッ」というサウンド（BLOCK_NOTE_BLOCK_IRON_XYLOPHONE 等）を再生。
### /vc broadcast <message...> (管理者用)
 1. **内部処理:** Djangoへ「放送開始通知」を送信。
 2. **表示:** サーバー全体にタイトルメッセージを表示。
## 4. 特殊仕様のフラグ管理（ロマン枠）
2kmのラジオやノイズ計算を豊かにするために、プラグインから追加で送るべきステータスです。
 * **is_sneaking:** スニーク中は声が少し小さくなる（またはノイズが増える）演出用。
 * **is_in_water:** 水中にいるときはボコボコとしたフィルターをかけるためのフラグ。
 * **y_coordinate:** 地下深くにいるかどうかの判定用。
## 5. 依存ライブラリの選定
plugin.yml または build.gradle に含めるべき主要ライブラリです。
 1. **Jedis:** Redis操作用（軽快でパイプライン処理に強い）。
 2. **Gson:** JSONの構築・パース用（Paperには標準で含まれています）。
 3. **OkHttp:** Django APIへのHTTPリクエスト用。
